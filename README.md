## 0. 프로젝트 개요 (간단 요약)

(※ 공통 과제이므로 1~2문단 이내로 간단히 작성)

본 프로젝트는 공통 요구사항을 기반으로 한  
칸반 형태의 태스크 관리 대시보드를 구현하는 과제입니다.

CRUD, 기간 필터, 통계, 테마 및 UX 요소를 포함하며  
팀 단위 협업을 통해 설계 및 구현을 진행했습니다.

### 3-2. 모듈 책임 분리

- createCard.js : 할일 추가 팝업에서 할일 입력시 할일 카드(할일 요소) 생성
- modalbtnfunction.js : 모달 내에서의 저장, 취소, 삭제 등등의 버튼 클릭시 실제 데이터 변경 및 삭제
- openmodalfunction.js : '새 할 일' 버튼 클릭시 할일 추가 모달, 카드 클릭시 할일 수정 모달, 카드 내 X 버튼 클릭시 카드 삭제 모달, 전체 데이터 초기화 버튼 클릭시 카드 초기화 모달을 띄우는 역할
- darkmode.js : 테마 설정 로컬스토리지 저장, 다크모드 토글 담당
- datareset.js : 전체 카드(할일 요소) 초기화 담당
- header.js : 헤더 닉네임 수정 및 로컬스토리지 저장, 실시간 인사말 및 현재 날짜 표시
- createFilterBadge.js : 사용자가 설정한 필터 상태를 화면에 배치
- filter.js : 설정한 필터 조건에 맞게 하단에 표시할 카드(할일 요소)를 골라내는 로직
- search.js : 검색어 입력시 실시간 필터링 이벤트
- app.js : 전체 화면 렌더링 진행, 필터 메뉴 클릭 상호작용 이벤트
- constant.js : HTML에서 가져올 요소 선택자, 로컬스토리지에 저장할 키 값, 필터 및 정렬에 사용되는 객체 모음집
- main.js : 로컬스토리지 데이터별 저장 및 불러오기 방식 전담, 렌더링 함수 로직

## 4. 핵심 설계 결정 사항 (Design Decisions)

- status 기반 보드 분리 → 상태 필터 제거
- 모든 날짜 데이터는 timestamp(number)로 통일
- 정렬 → 기간 → 우선순위 → 검색 순서의 고정 파이프라인 적용
- 통계는 기간 필터와 무관하게 전체 Todo 기준으로 계산
- 렌더링 함수를 통해 데이터와 화면 동기화
- 스크롤을 너무 내릴시 화면 최상단으로 올라갈수 있는 Go-Top 버튼 제공
- 실시간으로 검색 필터 입력값이 반영되는 디바운싱 검색 기능 제공

## 6. 트러블슈팅 (Troubleshooting)

### 6-1. 검색어 필터 적용 문제

- 증상: 검색어 필터 적용 후 데이터 초기화 시 카드 갯수가 실시간으로 갱신되지 않는 현상이 있었습니다.
- 원인: 검색어 필터 로직이 필터링 통합 로직으로 이동하지 않고 따로 적용되고 있었습니다.
- 해결: 검색어 필터링 로직을 통합 필터 로직으로 이동하여 해결했습니다.
- 회고: 필터링 로직을 따로 구성한 이후 통합 여부를 잘 확인해야함을 알게 되었습니다.

### 6-2. 할일 삭제 오류

- 증상: 하나의 할일을 삭제한 이후, 새로운 할일 추가시 TypeError가 발생했습니다
- 원인: 할일 삭제시 각 할일에 저장된 Id 값을 찾아서 삭제하는데, 삭제 이후 찾은 Id값을 초기화하지 않아서 수정 모드로 착각이 된 이후 존재하지 않는 Id값을 찾다가 에러가 발생했습니다
- 해결:삭제 이후 Id값을 null로 초기화하는 로직을 추가하는 방식으로 해결했습니다.
- 회고: 특정 값을 찾아서 데이터를 조작하는 방식을 사용할 경우에는 해당 로직이 끝난 이후 초기화 로직을 추가하는 습관을 들여야함을 알게 되었습니다.

### 6-3. 초기 아이디 값 설정 오류

- 증상: 아이디 값이 1로 초기화되어 아이디 값이 겹치는 상황이 발생하였습니다.
- 원인: 전역변수에 let count = 1을 하고 모달에서 객체를 생성하여 저장할 때마다 1을 증가시키는 방식을 택해 새로고침을 할 때마다 카운트가 1로 초기화가 되었습니다.
- 해결: 아이디는 고유한 값이어야 한다는 선생님의 말씀을 듣고 Date.now()를 통해 타임스탬프를 줘서 항상 고윳값을 줄 수 있게 변경하였습니다.
- 회고: 고윳값을 이용해야 한다면 Date.now()같은 타임스탬프값을 이용하는 것을 알게 되었습니다.

### 6-4. 필터 기능 구현

- 증상: 원하는 데이터 배열을 얻는 것은 성공하였으나, 카드 렌더링에 어려움을 겪었습니다.
- 원인: 렌더링 시에 출력되는 localStorage의 배열을 안 보이게하고, 원하는 조건만 출력하는 데에 있어 로직 구상이 막혔습니다.
- 해결: localStorage의 배열을 필터한 배열을 변수에 저장하고, 그 변수(배열)를 forEach문을 통해 카드를 렌더링하였습니다.
- 회고: 코딩테스트 문제를 풀며 배열 메서드에 좀 익숙해진 줄 알았지만, 아직 모자란다는 것을 깨닫고, 공부를 더 열심히 해야겠다고 생각했습니다.

## 7. 자체 평가 및 회고 (Self Review)

### 7-1. 잘한 점

- 역할 분담을 사전에 명확히 하고 작업하여 각자 그 기능을 집중하여 작업이 가능했습니다

- 팝업 등장 애니메이션 및 카드 이동 애니메이션을 구현하지 못한 점

- 인터섹션 옵저버를 통해 무한 스크롤 기능을 구현하면서 성능 최적화를 이루고 싶었으나 시간이 부족해서 진행하지 못한 점

### 7-3. 다음에 개선할 점

- 검색 기능에 자동완성 기능 추가
- 팝업 등장 애니메이션 및 카드 추가 및 삭제시 이동하는 애니메이션 추가
- 테마 토글 애니메이션 구현
